Act I · 02 / 08
The Paradigm Shift

解除对物理终端的强绑定

Before · 痛点与限制

孤立的本地 CLI 运行

  • 连接易断：电脑休眠或断网，长耗时 Agent 任务中途报废。
  • 产出难看：Agent 生成的 HTML/PDF，在终端黑窗口无法核对。
  • 多端受阻：外出时，在手机/平板上无法优雅唤起或监督 Agent。
After · 赋能解法

自适应 Web 智能工作空间

  • 进程持久：tmux 状态托管于 Go 后端，断线不终止，毫秒级接管。
  • 即时预览：文件管理新增 HTML/PDF 高清 16:9 全屏新页面预览。
  • 掌心控制：针对移动端大幅折叠键盘优化，直供 `claude` 极速指令。
让 Agent 的长时执行与可视化能力得到质的提升

Terminal Persistence · 终端持久化
Act I · 04 / 08
PERSISTENT WORKFLOW

别让你的 Agent 任务死于断线。

Claude Code 执行大规模代码修改、依赖检索和项目测试，往往需要持续数分钟。

Remote Agents 将 xterm.js 与 Go 守护监视的 ttyd/tmux 深度绑定。您可以在手机上语音唤起任务，合上浏览器，Agent 将在后台稳健推进。随时连回，进度完好如初。

"内置的守护进程让 tmux 会话获得了云端生命力，
每一次重新接管，都在 1 毫秒内瞬间发生。"
— 会话持久化机制
ttyd + tmux 会话生命周期由 Go 后端完全接管

Security Context · 局域网 HTTPS 部署
Act II · 06 / 08
SECURE BY DEFAULT

四步配置安全绿锁，解锁完整 Web API 🔒

高级 API（麦克风语音听写、Service Worker 缓存）强制要求安全上下文（HTTPS）。

自动生成与高级域名申请 · Setup Pipeline
01
启用 --ssl
启动参数中直接加 --ssl，后台若没有找到证书，会自动生成 10 年期高强度 ECDSA P-256 自签名证书。
02
Tailscale 自动识别
系统会自动检测是否运行了 Tailscale，将节点 IP 和域名（*.ts.net）自动注入证书 SubjectNames。
03
官方权威证书
亦可执行 tailscale cert 申请由 Let's Encrypt 签发的权威证书，将其移至默认 ~/.remote-agents/certs 目录。
04
全球无警告绿锁
重新运行直接适配，全球设备通过节点域名访问直接绿锁通过，完美支持浏览器原生麦克风录音权限！
支持局域网自签名与 Tailscale Let's Encrypt 的自适应加载
